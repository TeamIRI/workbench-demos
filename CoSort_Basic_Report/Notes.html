Simultaneous Data Transformation and Reporting in the File System

This project demonstrates several of CoSort's data transformation and reporting capabilities in a single SortCL job script, stockjoin.scl. A second job called mortgage.scl shows two more reporting features - derived field and math logic.

Note that both of these jobs were created prior to the IRI Workbench, and may not be the type of SortCL job the GUI user would enjoy creating. Nonetheless, the IRI Workbench accepts pre-existing, complex SortCL job scripts and allows you to modify and run them with section (e.g. /OUTFILE) and attribute (e.g. /FIELD-specific) GUI dialog editors.

NOTE: Master Data Management (MDM) and Customer Data Integration (CDI) reporting applications are also possible in SortCL, though purpose-driven wizards are not yet built into the GUI. End-users and consultants appreciate the flexibility that SortCL scripts provide for job design, maintenance, and external execution.

To use stockjoin.scl:

  1. Open both data sources from the project's InputFiles folder (nyse-a.txt, buys.csv) to see the formats and the field they share: the three-letter stock ticker symbol.
  2. Open the stockjoin.scl script, located in the scripts folder. It performs a sort and join on both input files in the action section and generates multiple output reports. These reports contain target-specific transforms, such as selection and aggregation. More than two data sources are possible, including relational tables connected via ODBC. Field-level protections could also be applied.
  3. Open the Targets/TradingA output file contents and layouts. They include the results of the full outer join, cross-calcs and the final aggregate.

The requirement to have 2-3 different (summary and detail) formats in targets like TradingA and the html version are common in custom SortCL BI reports; however, they are not intuitive for GUI users to write.

To use mortgage.scl:

  1. Open the following files:
    InputFiles/mortgage.dat
    scripts/mortgage.scl
  2. Right click the script and select Run As > IRI Job to see the output header format and detail (payment) values.
