Dark Data Discovery

This demonstrates the use of the Dark Data Discovery wizard for discovery. For discovery and remediation, along with an overview of the Named Entity Recognition (NER) capabilities, please refer to the DarkShield demo

This wizard searches information with specified formats using any combination of regular expression patterns and data classes. You can select an existing pattern from a pattern library, or create a new pattern to be stored in the library. A list of common patterns is available. You can also select Data Classes that are stored in your IRI preferences, which can be loaded with patterns, lookup set files, or NER models. The results of the search are organized into a usable format which can be loaded into a CoSort process with the help of a metadata file (ddf) that can be created in this wizard. If you include xls and/or xlsx document types in the search, an Excel® Interchange file (.eif), which can be opened and manipulated in Excel using CellShield®, is also created.

Before starting this demo, place all the documents to be searched in a single folder on your computer. The files for this demo are located in data folder.

Recommended steps to perform the demo in IRI Workbench:

Setup Page:

  1. Open the "New Dark Data Discovery" wizard from the Discovery Menu.
  2. In the Folder field, make sure Dark Data Discovery is the selected project folder.
  3. Select the check box to create a new folder using the name in the next field. The output files will be put into this folder.
  4. In the Name field, leave the default name of new_search for this example.
  5. Select the check box to create a DDF. This file will be put in the metadata folder.
  6. Click Next to continue to the next page.

Folder Selection Page:

  1. Click Select and use the directory tree view to navigate to the folder in the workspace called data.
  2. Click OK to add the folder to the list of search directories, and click Next to continue to the next page.

File Type Selection Page:

  1. In the File types section, click Select All or select the checkboxes for the file types which you wish to include in the search.
  2. Click Next to continue to the next page.

Metadata Selection Page:

  1. In the Metadata display field, select the type of information to be displayed in the report. For this example, Select All.
  2. In the Report Delimiter field, type the delimiter used to separate the fields of the metadata information in the generated report. You may leave the default.
  3. Click Next to continue to the next page.

Data Rules Page:

     Phone Number

  1. Click Add to add a new Data Rule Matcher.
  2. In the Matcher Name field, set the name you want to use for your matcher.
  3. In the Rule Name Field, click Browse and double-click on the DataRedactionRule to select it.
  4. In the Matchers field, click Add to open the Data Rule Matcher Details dialog.
  5. In the Details field, click Browse and double click on the PHONE_US pattern to add a US phone number pattern.
  6. Click OK to return to the Data Rule Matcher dialog, and click OK again to generate the new Data Rule Matcher.

     Emails

  1. Repeat steps 1-4.
  2. In the Type field, select Data Class from the dropdown.
  3. In the Details field, select EMAIL from the dropdown.
  4. In the Rule Name Field, click Browse and select the data rule you've created for the previous matcher.
  5. Click OK to return to the Data Rule Matcher dialog, and click OK again to generate the new Data Rule Matcher.
  6. Click Finish.

Execution:

  1. The wizard will generate a new file called new_search.search in the new_search folder of the Dark Data Discovery project.
  2. To run the search, right click on new_search.search and select run as -> IRI Search Job.
