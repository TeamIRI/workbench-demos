Dark Data Discovery

This demonstrates the use of the Dark Data Discovery wizard for discovery. For discovery and remediation, along with an overview of the Named Entity Recognition (NER) capabilities, please refer to the DarkShield demo

This wizard searches information with specified formats using any combination of Data Classes. You can also select Data Classes that are stored in your IRI preferences, which can be loaded with patterns, lookup set files, NER models, Image bounding boxes or facial detection/ recognition models. The results of the search are organized into a usable format which can be loaded into a CoSort process with the help of a metadata file (ddf) that can be created in this wizard. If you include xls and/or xlsx document types in the search, an Excel® Interchange file (.eif), which can be opened and manipulated in Excel using CellShield®, is also created.

Before starting this demo, place all the documents to be searched in a single folder on your computer. The files for this demo are located in data folder.

Recommended steps to perform the demo in IRI Workbench:

Setup Page:

  1. Open the New Dark Data Discovery Job wizard from the Discovery Menu.
  2. In the Folder field, make sure Dark_Data_Discovery is the selected project folder.
  3. Select the check box to create a new folder using the name in the next field. The output files will be placed into this folder.
  4. In the Name field, leave the default name of new_search for this example.
  5. Select the check box to create a DDF. This file will be put in the metadata folder.
  6. Click Next to continue to the next page.

Data Sources Page:

  1. Click Add to open the Data Source dialog. From here, you can select any folder on your file system. You can also select folders from an SMB share by selecting SMB in the Type field.
  2. In the Directory field, Click Browse and navigate to the data folder.
  3. Explain that in the Include field you can type in a regular expression pattern for which file names to include in the search. Alternatively, you can exclude certain files by adding a regular expression pattern in the Exclude field. Leave both blank for the demo.
  4. Select the recursive checkbox to recursively search through the selected folder.
  5. In the File Types field, click Select All to include all supported file types in the search.
  6. Click OK to add the folder to the list of Source URIs, and click Next to continue to the next page.

Data Targets Page:

  1. Click Next to skip this page, as we will not be remediating any documents.

Metadata Selection Page:

  1. In the Metadata display field, select the type of information to be displayed in the report. For this example, Select All.
  2. In the Report Delimiter field, type the delimiter used to separate the fields of the metadata information in the generated report. You may leave the default.
  3. Click Next to continue to the next page.

Search Matchers Page:

     Phone Number

  1. Click Add to add a new Search Matcher.
  2. In the Matcher Name field, set the name you want to use for your matcher.
  3. In the Data Class Name field, click Browse to select a Data Class or Data Class Group from the IRI preferences. In the dropdown box, select PHONE_US and click OK.
  4. In the Rule Name field, click Browse and double-click on the DataRedactionRule to select it. This will not be used in the search.
  5. Explain that in the Filters field, you may add filters to narrow down the search text based on file format specific filter statements. Currently, JSON path filters are available for json documents.
  6. Click OK to generate the new Search Matcher.

     Emails

  1. Repeat steps 1-4 from the previous section, this time selecting the EMAILS Data Class from the dropdown.
  2. Click OK to generate the new Search Matcher.
  3. Click Next to continue to the next page.
  1. Click Next to skip this page, as we will not be remediating any documents.

PDF Configuration Page:

  1. This page is only relevant in the DarkShield demo, and can be skipped. Click Finish to generate your new search job.

Execution:

  1. The wizard will generate a new file called new_search.search in the new_search folder of the Dark Data Discovery project.
  2. To run the search, right click on new_search.search and select run as -> IRI Search Job.
  3. You can examine the results of the search in the generated new_search.txt file, or using a more graphical interfaces by opening the new_search.darkdata file.
