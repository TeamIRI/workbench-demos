Generating Realistic Test Data in Custom File Formats Reports

In addition to structurally and referentially correct test data for databases, RowGen can create compliant, valuable test data with the same format and characteristics as data in production files and two-dimensional (2D) reports.

In the IRI Workbench, RowGen leverages both:

The CoSort Sort Control Language (SortCL)
data definition and manipulation/formatting program. RowGen uses the simple syntax and engine to filter, transform and map test data into one or more custom file and structured report formats at generation time. Targets can be files, pipes, ODBC or procedures. RowGen metadata is also ready for use in SortCL when testing shifts to production

Workbench Facilities
for project and metadata management, an IRI new job wizard, job outline, and dialog editors for test data creation and target file/report formatting in one or more RowGen (or SortCL) job scripts.

The two technologies combine into a purpose-built wizard for writing one or more test file targets in flat-file and/or structured report formats.

This project contains set file generation scripts and a report generation script. Set files are typically used in RowGen scripts as randomly-selected sources of real data. They populate columns in test files, or tables, ad hoc, or on a rule-specified basis. The sample .rcl files show that RowGen field data is generated or selected at random.

To create your own test files from scratch:

  1. Think about the file layout you are looking for and locate existing metadata (.ddf), if any. Otherwise, plan to build the new layouts one field at a time.
  2. Run the New Test Data Job wizard in the RowGen menu to create a new set file, flat file, or custom report. Follow the instructions and consult the in-context help. Or, use (and modify) an existing .rcl script example.
  3. Study and run your .rcl scripts (like examples already built in the script folder) and see the output and set files they produce. Each time you re-run these jobs different output values will be produced due to random generation (or selection).
  4. Edit the scripts in the editing windows or via Outline view by right-clicking on a source (generation) or target field and clicking Edit.
  5. You can re-use the set files you built in previous scripts inside new scripts.

To experiment further:

Use the New Set File wizard to create real data lists for random selection and population of field values in RowGen scripts. Wizards support:
  • data falling within numeric ranges or specified directly as Literal Values
  • real data in one or more columns in a database table (Set from Column)
  • real-looking data formats you construct on the fly as a Compound Data Value (e.g., add a parens, 3 whole numbers, parens, 3 whole numbers, dash, 4 whole numbers for US tel. area codes)

NOTE: You can choose the random selection method: ANY, ALL, ONCE, PERMUTE, SUFFIX, and ROW for sets. See the on-line documentation for descriptions of these options.
