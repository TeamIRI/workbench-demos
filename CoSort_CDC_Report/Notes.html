Data-based Change Data Capture (Delta Reporting)

This project demonstrates many of CoSort's data transformation and reporting capabilities in a single SortCL job script, CDC.scl. See below for more information:

www.iri.com/solutions/business-intelligence/embedded-bi/change-data-capture

NOTE: Master Data Management (MDM) and Customer Data Integration (CDI) reporting applications are also possible in SortCL, without purpose-built job wizards in the IRI Workbench. The ability to help customers build these kinds of reports manually for SortCL is therefore a services revenue opportunity for you, and a combined product offering that buyers of separate ETL and BI tools can appreciate for the savings in time, training, and money that SortCL represents. End-users and consultants appreciate SortCL scripting for its simple job design, modification, deployment and sharing.

To use CDC.scl:

  1. Open both input files (current.txt and previous.txt) in the sources folder to see the formats. Note the common fields between them.
  2. Examine the job script - CDC.scl:
    • input layouts
    • /JOIN statement
    • fields
    • condition logic -- this is used to display and select the various types of changes on the basis of the checksum field value
  3. Run the job by right-clicking on CDC.scl and selecting Run As > IRI Job, or click the green play button in the top menu.
  4. Open the targets folder and double click on TestDeltas.out to see how all forms of the changes are displayed in that file. It is also possible to output only the individual changes and format them similarly (as you can see in the other .out files), or in a custom way with other layouts and calculations that produce business intelligence from the comparisons.
  5. Clean up by deleting the 5 output files in the target folder.

As an advanced exercise, make additional output targets and/or new scripts to produce additional, customized reports.
