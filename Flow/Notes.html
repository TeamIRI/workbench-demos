Working with the IRI Flow and Database Models and Diagrams

The IRI Workbench now supports a work flow modeling tool and diagrams to help you design and build your data integration jobs. You will be working with the Sirius diagrams for Flow and ER models.

How to create a flow model

Set up your project:

  1. In the Project Explorer, right-click on your project. If this option is available, select Configure > Convert to Modeling Project.

Create a flow model in one of these two ways:

  1. Use the New Multi-Table Masking Job wizard to create a basic data integration flow. This will create a flow model and open the diagram.

OR

  1. To build an empty model to build to your preferences, select File > New > Flow.
  2. Select a project and click Finish This will open the diagram.

How to create an ER Diagram

  • Follow the setup instructions above.
  • First a database model must be created. To do this:
    1. In the Project Explorer, select the project.
    2. In the select Data Source Explorer, select the tables to be included in the diagram.
    3. Right-click and select New ER Diagram.
  • A database model is placed in the selected project location and the diagram is opened.

How to add elements to the diagrams

Select an element from the Flow Creation section of the Palette on the right and click in the diagram. The element is placed in the diagram. Use the connections in the Flow diagram to link items together.

By right-clicking in the Flow diagram you can get to the Flow popup menu. Here you can select various useful features. Note that some items are context sensitive so they will only show up at certain times.

How to open a mapping diagram

The mapping diagram allows you to see the mapping and transform in an aesthetically pleasing way.

Double click on a brown transform mapping block or a yellow FACT mapping block.
