Data Classification

This demonstrates the use of data classes and field rules.

Setup:

  1. In the Data Classes and Groups preferences page, confirm that these exist:

    • class “FIRST_NAME” that uses “C:/IRI/cosort100/sets/names/names_first.set” as a matcher
    • class “LAST_NAME” that uses “C:/IRI/cosort100/sets/names/names_last.set”
    • class “PIN_US” that uses “\b([0-9]{3}[-]?[0-9]{2}[-]?[0-9]{4})\b”
    • group “NAMES” with FIRST_NAME, LAST_NAME, and FULL_NAME selected
  2. Confirm that you do not have an existing iriLibrary.dataclass file in the project you are using.

Wizard:

  1. Right-click on the PERSONS table in the Data Source Explorer(DSE) and select IRI > New Data Classification Setup Job.
  2. On the setup page, enter the location to save the dataclass file. There can only be one of this file type in each project, hence the read-only status of the name field. Leave the checkbox selected. Click Next.
  3. The input page shows that one table is selected. Click Finish. This will create a data class library with the selected sources included. The form editor that opens allows you to classify the data in those sources.

Form Editor:

  1. Click on the data source to display details about that source. Leave the checkbox for including match via field name selected. Click Auto Classify and select SSN, LASTNAME, and FIRSTNAME. You should get data class results for those three fields (100%, 60% and 99%).

Wizard:

  1. Select the PERSONS table in the DSE and select IRI > New FieldShield Multi-Table Masking Job.
  2. On the setup page, leave loader blank so that a flat file is produced. Click Next. Click Next.

     Rule 1

  1. On the Field Modifications Rules page, click Create to open the Field Rule Matcher page. In the Matcher Name field, enter SSN. In the Rule Name field, click Create to open the field rule selection page.
  2. Select Redaction Function and click Next. Select the Define Mask radio button. Type 1 in Start position and 5 in Length. Click Add to Table and Finish.
  3. Click Add in the Matchers section. The Field Rule Matcher Details dialog opens. Fill dialog out with these details: Type: DATACLASS, Filter: blank, Details: PIN_US, Operator: AND. Click OK.
  4. On the Field Rule Matcher page, click OK.

     Rule2

  1. On the Field Modifications Rules page, click Create to open the Field Rule Matcher page. In the Matcher Name field, enter Name. In the Rule Name field, click Create to open the field rule selection page.
  2. Click Create and select Redaction Function. Leave Whole Field selected. Click Finish.
  3. Click Add in the Matchers section. Fill dialog out with these details: Type: DATACLASS, Filter: blank, Details: NAMES, Operator: AND. Click OK.
  4. Click Test and PERSONS.LASTNAME and PERSONS.FIRSTNAME should be returned. Click OK.
  5. Click Next and the summary screen should display the two rules. The first should include one matched column and the section with two matching columns. Click Finish.
  6. The script produced will show three fields being masked.
