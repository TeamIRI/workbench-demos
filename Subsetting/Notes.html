Subsetting Demo

This demo shows how to create a smaller, referentially-correct copy of a larger database.

This demo requires and uses the following resources:

  • The DTP connection profile Oracle
  • The ODBC DSN Oracle

IMPORTANT: Before running this demo, if the *_DIM tables do not exist, expand the setup folder and right-click reset_demo.bat. Select Run As > Batch Program. If they do exist, expand the setup folder and right-click truncateTables.sql.

Steps

  1. Select the Subsetting project in g>Project Explorer.
  2. In Data Source Explorer select Oracle > XE > Schemas > SCOTT > Tables.
  3. Right click the source table SALE_DIM and select IRI > New Subset Job.
  4. On the Define Destination page, select the loader ODBC. Click Next.
  5. The SALE_DIM table is already selected on the Data Extraction Stage page so click Next.
  6. The Sort page is where you specify the sort order of the subset. For example, to get a subset of the highest quantity sold levels, you would sort on the quantity sold field in descending order (as in this example). If you want the rows selected randomly, however, leave the Key Fields list blank. Double click the QTY_SOLD field. Click the key on the right and then Edit Key and select Descending next to Direction. Click OK. Click Next.
  7. The Options page is where you specify the size of the subset and any filters. The size and filter is applied to the source table. Enter 100.
  8. The Data Loading page will display if you have selected a loader. It has two functions. The first is to name the output targets for the subset. The second allows you to view the relations of your source table.
  9. Specify the Oracle Connection Profile and the SCOTT Schema Name..
  10. Under Naming options define the details of the target tables. The tables are loaded back into the same schema so the table names must be changed so the source table is not overwritten. Select the Prefix option, enter SUB_ and click Set.
  11. In the lower section of the page, the new names of the tables have been prepended with the prefix. Selecting a different table in Original will display the new table name. Click Next.
  12. On the Field Modifications Rules page, you can enter rules to apply to the tables. Care must be taken not to apply a rule to a foreign key which would impact the results.
  13. On the Summary page, you can review the details of your job. Click Finish.

Cleanup

  1. Delete the new_subset folder.
  2. Execute truncateTables.sql in the setup folder.
