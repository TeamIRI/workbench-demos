Data Vault Generator with random data

This demonstrates the use of the Data Vault wizard to create a data vault from existing tables and loading with randomly generated data.

This demo requires and uses the following resources:

  • The DTP connection profile Oracle
  • The ODBC DSN Oracle

IMPORTANT: Before running this demo, expand the setup folder and right-click reset_demo.bat. Select Run As > Batch Program.

Recommended steps to perform the demo in the Workbench:

Steps

  1. Open the document DV.sqlschema to see the existing table relations.
  2. Open the Oracle connection in the Data Source Explorer.
  3. Navigate to the XE > Schemas > SCOTT > Tables.
  4. Select the seven tables for the example: DV_COUNTRIES_HR, DV_DEPARTMENTS_HR, DV_EMPLOYEES_HR, DV_JOBS_HR, DV_JOB_HISTORY_HR, DV_LOCATIONS_HR, and DV_REGIONS_HR.
  5. Right-click the tables and select IRI > Data Vault Generator
  6. On the first page, make sure that the main project folder is specified.
  7. Select Load randomly generated data as the Output. Leave the Hash type as MD5. Select ODBC as the Loader. Enter 1000 in Rows to generate. Select Include a last seen date column for hubs and links. Select Include a has diff column for satellites. Select Show summary page. Click Next
  8. On the Table Selection page, the seven tables should already be pre-selected. Click Next.
  9. On the Hub Options page, in the Columns column, double-click the COUNTRY_ID row. On the details page, unselect COUNTRY_ID and select COUNTRY_NAME.
    Double-click the DEPARTMENT_ID row. On the details page, unselect DEPARTMENT_ID and select DEPARTMENT_NAME.
    Double-click the JOB_ID row. On the details page, unselect JOB_ID and select JOB_TITLE.
    Double-click the REGION_ID row. On the details page, unselect REGION_ID and select REGION_NAME. Click Next.
  10. On the Link Options page, leave the defaults. Click Next.
  11. On the Satellite Options page, leave the defaults. Click Next.
  12. On the Data Loading Stage page, select the connection profile Oracle and schema SCOTT for the target. If there is only one schema, the wizard will auto-fill it.
  13. Leave APPEND in Output mode.
  14. Select Include a drop table script. Click Next.
  15. On the rules page, select Include rules associated with data classes at the bottom of the page. Click Next.
  16. On the Summary page, the list of the 20 new tables will be shown. Click Finish.
  17. A new folder, new_dataVault, will be created.

Output

  1. An ER diagram showing the relationships of the new tables will open.
  2. Also included is a new_dataVault.flow file. Double-click to open it and see the 27 transform mapping blocks that comprise the job.
  3. Right-click the new_dataVault.bat file and select Run As > Batch Program.

Cleanup

  1. Delete the new_dataVault folder.
  2. Execute reset_demo.bat in the setup folder.
