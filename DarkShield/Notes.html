Finding and Masking PII in Unstructured Text Files using DarkShield

This project demonstrates DarkShield's capabilities by taking you through the steps of creating and modifying a new Dark Data Discovery job to search for and remediate names, emails, and SSNs in unstructured file formats using regular expression pattern matching and Named Entity Recognition (NER). Notes2 of this project covers training a custom NER model, and is considered optional depending on the requirements and time constraints.

To use the DarkShield remediation capabilities, this demo requires a licensed installation of CoSort/SortCL.

Recommended steps to perform the demo in IRI Workbench:

NER Model:

For the following example, you will use a NER model downloaded from the OpenNLP website.

  1. Navigate to the OpenNLP models website in your browser.
  2. Explain the different NER models that can be downloaded and used, including ones for persons, organizations, and locations, and for different languages. Valid NER models have the Name Finder Component listed next to them.
  3. Explain that IRI provides additional means of training your own NER models if the pre-trained models provided on the OpenNLP website are not accurate for the client's documents.
  4. Download the en-ner-person.bin English Person Name Finder, and save it in a location where it can be easily found.

Dark Data Discovery Wizard

  1. Open the New Dark Data Discovery Job wizard from the Discovery menu.
  2. In the Folder field, make sure DarkShield is the selected project folder.
  3. Select the check box to create a new folder using the name in the next field. The output files will be placed into this folder.
  4. In the Name field, leave the default name of new_search for this example.
  5. Select the check box to create a DDF. This file will be put in the metadata folder.
  6. Click Next to continue to the next page.
  7. Click Select and use the directory tree view to navigate to the folder in the workspace called data.
  8. Click OK to add the folder to the list of search directories, and click Next to continue to the next page.
  9. In the File types section, click Select All or select the checkboxes for the file types which you wish to include in the search. Only a subset of these file types may be remediated depending on the formats that DarkShield currently supports.
  10. Click Next to continue to the next page.
  11. In the Metadata display field, select the type of information to be displayed in the report. For this example, Select All.
  12. Click Next to continue to the next page.

Names Matcher

  1. Click Add to open the Search Matcher dialog.
  2. In the Matcher Name field, type "NamesMatcher".
  3. In the Data Class Name field, click Create to open the Data Class or Group dialog.
  4. In the Name field, type "NAMES_NLP".
  5. In theMatchers field, click Add to open the Data Class Matcher dialog.
  6. In the Type field, select NER model.
  7. Click Browse to navigate to the downloaded NER model file and click OK.
  8. Click OK to add the Data Class Matcher to the NAMES_NLP Data Class.
  9. In the Rule Name field, click Browse and select the EncryptNamesRule from the list.
  10. Click OK to add the NamesMatcher to the list of Search Matchers.

Emails Matcher

  1. Click Add to open the Search Matcher dialog.
  2. In the Matcher Name field, type "EmailsMatcher".
  3. In the Data Class Name field, click Browse to open the Data Class Selection dialog.
  4. In the Data Class field, select the EMAILS Data Class and click OK.
  5. In the Rule Name field, click Browse and select the HashEmailsRule from the list.
  6. Click OK to add the EmailsMatcher to the list of Search Matchers.
  7. Click Finish.
  8. The wizard will generate a new file called new_search.search in the new_search folder of the DarkShield project.

Results:

  1. To run the search & remediation, right click on new_search.search and select run as -> IRI Search and Remediate Job.
  2. In the Project Explorer view, navigate to the DarkShield -> search_output folder.
  3. Double click on the search_output.txt file to open the results file in the Workspace.
  4. In the Project Explorer view, navigate to the new_search-remediated\path\to\workbench-demos\DarkShield\data folder.
  5. Double click on any of the files within the folder to open up the remediated file in the Workspace.
  6. For formats with no built-in Workbench editors (like pdfs), right click on the file and select Open with -> System Editor to open the file in a separate application.
  7. When opening a remediated pdf file, explain that DarkShield currently only supports full text redaction regardless of the specified rule.
  8. Double-click on the new_search.darkdata file to open the DarkData form editor. You can click on individual documents to expand the list of search results found in each document.
  9. Right-click on the new_search.darkdata file and select IRI -> Graph to open a graph of the results.
  10. If the graph did not open up correctly, close the window, right click on the new_search-graph.html file and select Open with -> Web Browser.
  11. Explain that this graph shows the search and remediation statistics on the scanned documents, such as the distribution of file types containing the search results, and a risk assessment from how many files are protected in each format category. A file is only considered protected within this chart if every search result within it was remediated.
