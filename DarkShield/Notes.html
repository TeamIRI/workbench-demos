Finding and Masking PII in Unstructured Text Files

This project demonstrates most of the DarkShield's capabilities by taking you through the steps of searching for Personally Identifiable Information (PII) in unstructured file formats, recording the results and file metadata for further analysis using IRI's structured data tools (CoSort/sortCL), and remediating the PII from the original document.

To take full advantage of IRI Workbench's data reporting capabilities, this demo requires the installation of CoSort/SortCL (which runs .scl scripts) and the Business Intelligence Reporting Tool (BIRT), which is available through their Eclipse update site.

Recommended steps to perform the demo in IRI Workbench:

Wizard:

  1. Open the "Dark Data Discovery" wizard from the Discovery menu.
  2. In the Folder field, make sure DarkShield is the selected project folder.
  3. Select the check box to create a new folder using the name in the next field. The output files will be put into this folder.
  4. In the Name field, leave the default name of search_output for this example.
  5. Select the check box to create a DDF. This file will be put in the metadata folder.
  6. Select the check box to remediate. This will create a remediated copy in the output folder.
  7. Click Next to continue to the next page.
  8. In the Search directories section, click on the Select button and navigate to the folder in the workspace called data.
  9. Select the following file extensions in the File type extensions sections: txt, json, pdf, xml. At this time only those files are able to be remediated.
  10. Select All for the Metadata display sections.
  11. Click Next to continue to the next page.
  12. Select Create -> Browse Common Patterns and double click on Email from the table to select the pattern.
  13. Click Ok.
  14. Click Add to table to add the Email pattern to the search criteria.
  15. Repeat the previous three steps, this time selecting the PIN US pattern.
  16. Optionally, click Browse to add patterns from the IRI Patterns Library for the project.
  17. Click Finish.
  18. Wait for the wizard to generate the results.

Results:

  1. In the Project Explorer view, navigate to the DarkShield -> search_output folder.
  2. Double click on the search_output.txt file to open the results file in the Workspace.
  3. In the Project Explorer view, navigate to the saved-example-remediated\git\workbench-demos\DarkShield\data folder.
  4. Double click on any of the files within the folder to open up the remediated file in the Workspace.
  5. For formats with no built-in Workbench editors (like pdfs), right click on the file and select Open with -> System Editor to open the file in a separate application.

Report:

  1. From the Window menu, select Open Perspective -> Other -> Report Design.
  2. Click OK.
  3. Locate and open DarkDataResults.rptdesign. This is a graph created from DarkDataReport.scl, which counts the number of file types scanned during the discovery process.
  4. On the toolbar, click the View Report icon to run the report.
