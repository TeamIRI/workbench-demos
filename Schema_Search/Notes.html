Schema Pattern Search

This demonstrates the use of the schema pattern search and pattern search data class association.

Schema Pattern Search Wizard:

The Schema Pattern Search will create three text files. One is tableSearchResults.log that records all tables that are successfully searched. The columnSearchResults.csr records all columns that contain data that match the patterns. The schema_pattern_search.txt (or custom named file on the setup page) contains the column name and data that matched.

  1. Right-click on the SCOTT schema in the Data Source Explorer(DSE) and select IRI > Schema Pattern Search .
  2. On the Setup page, enter the location to save the result files. Click Next.
  3. The Schema Selection page shows that one schema is selected. Click Next.
  4. The Data Type Selection page is where you select the type of data to search. The text-type data types are pre-selected. Leave the defaults. Click Next.
  5. The Pattern page is where the patterns to be used are added. Click Create and then Browse Common Patterns. Double-click CREDIT_CARD at the top and then OK.
  6. Click Finish.
  7. The search results are displayed.

Associate Pattern Search to Data Class Wizard:

  1. Right-click on columnSearchResults.csr and select IRI > Associate Pattern Search to Data Class.
  2. On the Setup page, leave the defaults. Click Next.
  3. The Data Class Selection page is where the data classes are assigned. In the Data Class drop down next to SCOTT.PERSONS.CREDIT_CARD, select CREDIT_CARD. Click Finish.

Form Editor:

  1. This will create/append a data class library with the selected sources included. The form editor that opens also includes the data class and map for the selection in the wizard.
  2. Next to CREDIT_CARD in the Field Rule drop down, select the MaskingRule. Now, this rule will be available every time this data source is used in the Multi Table wizards.
